The experiments were conducted on synthetically generated random geometric undirected graphs. For each graph size (5\,000, 10\,000, 20\,000, 50\,000, 100\,000, and 250\,000 vertices), the average vertex degree was fixed at 7, resulting in graphs of similar density while varying only the number of vertices. Each graph was partitioned into different numbers of storage nodes (4, 6, 8, 10, 12 and 16) using the Fennel algorithm.

The primary evaluation metric was the graph cut percentage, defined as the percentage of edges whose endpoints belong to different partitions. A smaller graph cut corresponds to less inter-partition communication during graph traversal.

Each graph was processed incrementally. Vertices were inserted sequentially using the Fennel partitioning algorithm. After every 1\,000 inserted vertices, the current partitioning was refined using the Kernighan--Lin (KL) algorithm. This interval was selected as a compromise between partition quality and computational overhead. The graph cut was measured after the entire graph had been processed. For comparison, the same graphs were also partitioned using Fennel without any refinement.

Figure \ref{fig:experiments_res} shows that KL refinement consistently reduces the graph cut for all evaluated graph sizes. Although the relative improvement decreases for larger graphs, the reduction in graph cut remains significant.

An additional observation concerns the convergence of the KL algorithm. For all evaluated graph sizes, the algorithm typically converged after only two refinement iterations for each pair of partitions. Only a small number of cases required three iterations, and no experiment required more than three. This supports the observation reported in the original METIS work that a good initial partitioning substantially reduces the amount of refinement required \cite{MetisOG}.

The experiments were performed only on synthetic graphs of modest size. Evaluating the proposed approach on real-world graph datasets and larger graphs remains part of future work.

These observations indicate that Fennel produces an initial partitioning that is sufficiently close to a locally optimal solution, allowing KL refinement to converge rapidly while still reducing the graph cut.
