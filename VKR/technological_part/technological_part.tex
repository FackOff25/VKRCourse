\section{Технология тестирования} \label{technological_part}

Для оценки эффективности разработанной системы было проведено два набора вычислительных экспериментов. В обоих случаях в качестве основной метрики использовалось количество межшардовых переходов, возникающих при выполнении запросов поиска пути. Данная метрика позволяет оценить качество распределения данных и степень локальности размещения связанных вершин.

Первый набор экспериментов был направлен на исследование влияния количества хранилищ на качество распределения графа. Для каждого рассматриваемого числа шардов выполнялась следующая последовательность действий:
\begin{enumerate}
\item генерировался граф заданного размера;
\item формировался набор запросов поиска пути между вершинами графа;
\item создавалась распределённая среда с заданным количеством хранилищ;
\item выполнялась потоковая загрузка вершин графа в систему;
\item после добавления каждых 500 вершин запускалась процедура оптимизации распределения данных;
\item после завершения загрузки графа выполнялся подготовленный набор запросов;
\item после обработки каждых 100 запросов запускалась процедура повторной оптимизации распределения;
\item для каждого выполненного запроса фиксировалось количество межшардовых переходов;
\item после завершения серии запросов результаты сохранялись для последующего анализа.
\end{enumerate}

Описанная процедура повторялась для нескольких вариантов количества хранилищ, что позволяло исследовать влияние степени фрагментации данных на эффективность выполнения запросов и качество работы алгоритмов распределения и оптимизации.

Второй набор экспериментов был посвящён исследованию влияния неравномерного распределения вычислительных ресурсов. В данном случае количество хранилищ фиксировалось и составляло четыре шарда, однако размеры отдельных хранилищ различались между собой. Последовательность выполнения эксперимента была следующей:

\begin{enumerate}
\item генерировался граф заданного размера;
\item формировался набор запросов поиска пути;
\item создавалась распределённая среда, состоящая из четырёх хранилищ различной ёмкости;
\item выполнялась потоковая загрузка вершин графа;
\item после добавления каждых 500 вершин запускалась процедура оптимизации распределения;
\item после завершения загрузки выполнялась серия запросов поиска пути;
\item после обработки каждых 100 запросов производилась повторная оптимизация распределения;
\item выполнялся сбор статистики по количеству межшардовых переходов;
\item полученные результаты сохранялись для дальнейшего сравнения.
\end{enumerate}

После завершения всех серий экспериментов накопленные данные использовались для построения графиков зависимости количества межшардовых переходов от этапа выполнения эксперимента. Полученные визуализации позволяли сравнить различные конфигурации распределённой системы и оценить влияние числа хранилищ, их ёмкости и процедур периодической оптимизации на эффективность обработки запросов.
